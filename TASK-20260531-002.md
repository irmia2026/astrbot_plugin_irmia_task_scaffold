## 需求

dashboard.html 每 3 秒 `setInterval` 调用 `loadCurrent()` + `loadActivity()`，
每次都无差别重绘整个 DOM（`innerHTML = _skelTask()` + 重建表格/活动流），
导致进场动画反复重置、进度圈重新填充、视觉效果闪烁。

## 根因

│ `loadCurrent()` 第一行就清空 tbody → 骨架屏闪现
│ `loadActivity()` 无条件拼接 innerHTML → 直播流重绘
│ 没有数据比对——即使数据完全相同也全量重建

## 改动

（以下改动已由弥亚直接执行，此单归档备案）

### dashboard.html

| 位置 | 改动 |
|------|------|
| `let ST={...}` | 新增 `_cacheCur:'', _cacheAct:''` 缓存字段 |
| `loadCurrent()` | 先 `get('/current')`，`JSON.stringify` 比对 `_cacheCur`，相同则 `return`（跳过 DOM 操作）；不同才更新缓存并重建 UI |
| `loadActivity()` | 同理，先 `get('/activity')`，`JSON.stringify` 比对 `_cacheAct`，相同跳过 |

### dashboard-standalone.html（离线快照）

│ 同步以上缓存逻辑
│ `setInterval` 改为条件执行：离线模式（`API==='__EMBEDDED__'`）完全跳过轮询
│ 纳秒栏显示「离线快照」替代实时时间

## 不改动

- 轮询间隔（保持 3000ms）— 间隔本身不是问题，无差别重绘才是
- 骨架屏机制 — 首次加载保留，仅跳过重复渲染
- metadata.yaml / main.py — 无关
